Fusionne 3 axes : objectif cadre la cible+ton / signal "bon moment" justifie le timing / Plume tranche et explique. _(Axe "miner les dormants pour eux-mêmes" d'abord écarté — voir D2-bis qui le rouvre.)_

- **D5 — Data externe "bon moment" = enrichment auto** (rejeté : le diff CSV manuel, jugé pas assez auto). Fournisseur de départ = **People Data Labs** (gratuit 100/mo, $0.28/match, RGPD défendable).

- **D7 — Tension privacy (vs NFR-4 / project-context "zéro partage tiers").** MVP = PDL pour le founder seul (validation). Garde-fou marque = "privacy par défaut, enrichissement = opt-in explicite". - **D8 — Horizon = dogfood founder + SaaS.** MVP validé sur le founder d'abord.

## Décisions prises pendant la rédaction du brief

- **D9 — Problème affiné (2026-06-22).** Le mal n'est PAS la paralysie/inaction. Founder agit déjà, mais *agit mal* : (a) rate le bon moment, (b) part au hasard sans stratégie liée à un objectif → outreach gaspillé + fenêtres ratées. Campagne = problème de **précision**, pas d'inaction.
- **D10 — North star = composite "réponse × timing".** Une mesure : taux de réponse des messages issus de bons moments captés, vs baseline (hasard). Métriques secondaires : réactivation de dormants, adhésion (actions/jour tenues). Écarté comme principal : taux de réponse seul, fenêtres captées seul, adhésion seule (mesure l'usage pas la valeur). NB : remplace SM-1 (distance d'édition, caduc en conversationnel).
- **D2-bis — RÉOUVERTURE de D2 (2026-06-22).** Les contacts dormants REVIENNENT dans le concept (founder les veut). L'axe "réveiller les dormants", écarté en D2, est réintégré.
- **D11 — Cadrage dormant CONFIRMÉ (2026-06-22).** Un dormant remonte s'il est lié à l'objectif actif **OU** porteur d'un signal de timing (pas "ET" → ne pas rater les dormants pertinents sans signal ; pas "tous" → éviter la liste froide brute = Epic 4).
- **D12 — Définition "bien timé" CONFIRMÉE (2026-06-22).** Message bien-timé = parti dans les N jours d'un signal détecté. N = paramètre à fixer au PRD (candidats 7/14). North star (D10) se calcule sur cette définition.
- **D13 — Frontière scope MVP CONFIRMÉE (2026-06-22).** Campagne opère sur le **réseau existant only** (contacts déjà dans Plume + enrichment dessus). Sourcing net-new externe (recherche d'inconnus type LinkedIn) = OUT (risque légal, horizon SaaS lointain).
- **D14 — Signaux v1 dogfood (2026-06-22).** IN : (a) changement de poste via PDL, (b) pertinence vs objectif actif (scoring LLM), (c) froideur + relance en suspens (data interne gratuite, zéro API). OUT → v2 : **news par boîte** (levée/recrutement) — 2e source externe (API news + polling par entreprise), signal plus faible que le job-change, déféré pour garder le MVP serré. _(Founder a d'abord coché news en v1, puis corrigé : defer.)_
- **D15 — Une campagne active à la fois (v1) (2026-06-22).** Un seul objectif actif au v1. Multi-campagnes parallèles (job + freelance simultanés) = différé. Hypothèse de l'assistant, confirmée par le founder.
- **D16 — Persona gardé large (2026-06-22).** Primaire concret = le founder (chercheur d'emploi via réseau, dogfood). Secondaire = tout "networker actif" (chercheur d'emploi / freelance / founder-sales solo), volontairement non resserré à ce stade ; à trancher au PRD si SaaS.
